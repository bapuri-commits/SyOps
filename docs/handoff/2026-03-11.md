## 작업 내용
- **VPS 첫 배포 완료**
  - `/opt/apps/syops` 클론, backend venv, frontend 빌드
  - `/opt/envs/syops.env` 생성 (비밀번호 + 시크릿 키), 심링크
  - `syops.service` systemd 등록 + enabled (포트 8300, localhost only)
  - nginx 설정 교체: `syworkspace.cloud` → SyOps 포털 (정적 + API 프록시)
- **Node.js v22 설치** (NodeSource): Ubuntu 기본 v18 → v22.22.1 업그레이드 (Vite 7 요구)
- **nginx 보안 헤더 추가**: `security-headers.conf` 스니펫 생성, 전 서비스 블록에 include
  - X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, HSTS, Referrer-Policy
- **CI/CD 파이프라인 활성화**
  - GitHub Secrets 설정 (VPS_HOST, VPS_USER, VPS_SSH_KEY)
  - sudoers 설정: `/etc/sudoers.d/deploy` — dev 계정 passwordless sudo for systemctl restart/reload
  - 빈 커밋 → push → frontend ✅ backend ✅ deploy ✅ 전체 그린
- **pyproject.toml 수정**: `[tool.setuptools] packages = []` — flat-layout discovery 에러 해결
- **문서 갱신**: vps-server.md (SyOps 서비스, Node.js, 보안 헤더 반영), PLAN.md (Phase 1~4 완료 표시)

## 핵심 결정 & 맥락
- **sudoers 배포 권한**: CI/CD SSH에서 `systemctl restart syops`가 비밀번호 요구로 실패 → `/etc/sudoers.d/deploy`에 NOPASSWD 규칙 추가
- **Node.js 22 (NodeSource)**: Ubuntu 24.04 기본 Node 18은 Vite 7/Tailwind v4와 호환 불가
- **pyproject.toml flat-layout**: backend 디렉토리에서 `pip install -e .` 시 core/routers/services가 최상위 패키지로 인식되는 문제 → `packages = []`로 해결
- **gh CLI 미인증**: winget으로 설치했으나 브라우저 인증 필요. GitHub Secrets는 웹 UI에서 수동 설정
