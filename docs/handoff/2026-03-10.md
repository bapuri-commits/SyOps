# SyOps Handoff — 2026-03-10

## 현재 상태
PLAN.md의 4단계 로드맵(Phase 1~4) 전체 구현 완료. 코드는 커밋+푸시됨. VPS 배포는 아직 미실행.

## 작업 내용
- **Phase 1**: React + Vite + Tailwind v4 통합 랜딩 페이지 (`/`)
  - 서비스 카드 (QuickDrop, News Agent), 상태 뱃지, 반응형 디자인
- **Phase 2**: FastAPI 서비스 상태 API
  - `GET /api/health` (공개), `GET /api/metrics` (인증), QuickDrop HTTP 체크, News Agent 파일 체크, nginx systemctl 체크, psutil 메트릭
- **Phase 3**: 관리 대시보드 (`/dashboard`)
  - 로그인 → 실시간 CPU/RAM/Disk 게이지 (10초 폴링), SSL 인증서 만료 표시, 서비스 로그 조회/재시작, 디스크 경고 배너
  - 추가 API: `GET /api/ssl`, `GET /api/services/{id}/logs`, `POST /api/services/{id}/restart`
- **Phase 4**: CI/CD + 배포 자동화
  - `.github/workflows/ci-cd.yml`: frontend build + backend lint → deploy (SSH) → health check
  - `deploy/systemd/syops.service`, nginx 포털 설정, deploy.sh 업데이트
- **강도 높은 구현 감사 4회** 수행, 총 24건 이상 이슈 발견 및 수정
  - 소켓 누수, 401 세션 stuck, systemd WorkingDirectory 불일치, 프론트엔드 VPS 미전달, Unicode btoa crash 등 치명적 버그 사전 수정

## 핵심 결정 & 맥락
- **Node.js 설치**: 윈도우에 winget으로 Node.js v24 LTS 설치 (이전에 없었음)
- **Tailwind v4**: `@tailwindcss/vite` 플러그인 방식. `@theme` 디렉티브로 커스텀 색상
- **인증 방식**: HTTP Basic Auth (sessionStorage 저장). 개인 프로젝트 수준으로 충분
- **SSL 캐싱**: 인메모리 1시간 TTL. 매 대시보드 진입마다 소켓 연결 방지
- **CI/CD 통합**: 별도 ci.yml + deploy.yml → 단일 ci-cd.yml 병합. deploy가 CI 성공에 의존
- **systemd WorkingDirectory**: `/opt/apps/syops` (프로젝트 루트). relative import를 위해 backend/ 아닌 상위 디렉토리 필요
