# SyOps Handoff — 2026-03-12

## 현재 상태
블록 1(Docker화 + 서비스 파이프라인 표준화) 완료. QuickDrop이 Docker 컨테이너로 전환됨.

## 작업 내용
- Docker 29.3.0 + Compose 5.1.0 VPS에 설치
- QuickDrop을 systemd → Docker 컨테이너로 전환 (파일럿)
- 서비스 추가 표준 템플릿 생성 (`deploy/templates/`)
  - Dockerfile.fastapi, docker-compose.service.yml, nginx-service.conf, README.md
- SyOps 모니터링 Docker 연동
  - `docker_svc.py` 신규 생성 (컨테이너 로그/재시작)
  - `systemd.py`를 통합 레지스트리로 확장 (ServiceType enum + ServiceEntry dataclass)
- `deploy.sh` Docker 기반 배포 함수 추가
- QuickDrop CI/CD Docker 배포 워크플로 추가 (`.github/workflows/deploy.yml`)
- sudoers `/etc/sudoers.d/deploy` 문법 오류 수정 (CRLF 오염 파일 제거 + 재생성)
- cron cleanup → `docker exec` 방식으로 전환
- 감사 후 수정: Dockerfile 템플릿 포트 고정, dead code 제거

## 핵심 결정 & 맥락
- **nginx는 호스트에 유지**: SSL 종료 + 리버스 프록시 역할. 컨테이너화 불필요.
- **SyOps 백엔드도 호스트에 유지**: systemctl/journalctl + psutil 호스트 메트릭 접근 필요. 관리 플레인은 호스트가 실용적.
- **나머지 서비스만 Docker 컨테이너화**: 표준 파이프라인으로 추가.
- **바인드 마운트 사용**: 기존 `/opt/data/quickdrop/` 경로 유지, named volume 대신 bind mount로 데이터 연속성 보장.
