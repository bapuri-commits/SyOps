# Gallery 서비스 — 사진/영상 뷰어

> 2026-03-13 작성. 웹개발 연습 목적의 미디어 갤러리 서비스.

## 개요

- **목적**: 웹개발 기초 연습 (React 컴포넌트, API 호출, 파일 서빙)
- **기능**: 사진/영상을 업로드하고 갤러리 형태로 열람
- **모바일 우선**: 모바일 호환성 최우선 설계

---

## 접근 방식

SyOps 포털에 `/gallery` 경로로 통합 (방법 A).

- 별도 프로젝트/서브도메인 없이 기존 SyOps React + FastAPI 인프라 활용
- 로그인, Navbar 등 기존 컴포넌트 재사용
- 나중에 규모가 커지면 별도 서브도메인 분리 가능

## 구현 단계

### Phase 1 — 기본 갤러리 (최소 기능)

**백엔드** (`backend/routers/gallery.py`):
- `GET /api/gallery` — 파일 목록 JSON 반환
- `GET /api/gallery/files/{filename}` — 파일 서빙 (FileResponse)

**프론트엔드** (`frontend/src/pages/Gallery.tsx`):
- 파일 목록 fetch → 이미지/영상 구분해서 격자 표시
- `<img>` / `<video>` 태그로 렌더링
- Tailwind CSS 반응형 그리드

**라우팅**:
- `App.tsx`에 `/gallery` 경로 등록

### Phase 2 — 업로드 + 개선

- 브라우저에서 파일 업로드 (`POST /api/gallery/upload`)
- 썸네일 자동 생성 (Pillow/ffmpeg)
- 라이트박스 (사진 클릭 시 확대 뷰)
- 영상 스트리밍 최적화

### Phase 3 — 부가 기능

- 폴더/앨범 분류
- 정렬/필터 (날짜, 타입)
- 삭제 기능
- 드래그앤드롭 업로드

## 모바일 호환성 — 주의사항

### 1. 반응형 레이아웃

- **Tailwind 브레이크포인트 활용**: `grid-cols-2 md:grid-cols-3 lg:grid-cols-4` 식으로 화면 크기별 컬럼 수 조절
- **모바일 퍼스트**: 기본 스타일을 모바일용으로 작성하고, `md:`, `lg:` 접두사로 큰 화면 대응
- **뷰포트 메타 태그**: `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수 (Vite 기본 포함)

### 2. 이미지 최적화 (가장 중요)

- **원본 그대로 내려보내지 않기**: 5MB짜리 사진을 모바일에서 수십 장 로드하면 데이터 폭탄 + 렌더링 느림
- **썸네일 생성**: 갤러리 목록에서는 작은 썸네일(300~500px), 클릭 시 원본 로드
- **`loading="lazy"`**: 화면에 보이는 이미지만 먼저 로드, 스크롤하면 나머지 로드
- **적절한 이미지 포맷**: WebP가 용량 대비 화질 최고. 브라우저 호환성도 이제 충분

### 3. 영상 재생

- **`<video>` 코덱 호환**: MP4(H.264)가 모든 모바일 브라우저에서 가장 안전. WebM은 iOS Safari에서 제한적
- **`playsinline` 속성 필수**: iOS Safari에서 이거 없으면 영상이 전체화면으로 강제 전환됨
- **`preload="metadata"`**: 영상 전체를 미리 다운로드하지 않고 메타데이터(길이, 썸네일)만 로드
- **큰 영상은 스트리밍**: Range 요청 지원 (FastAPI의 StreamingResponse) — 안 하면 영상 전체 다운로드 후 재생

### 4. 터치 인터랙션

- **터치 타겟 최소 44x44px**: Apple/Google 가이드라인. 버튼이나 터치 영역이 너무 작으면 손가락으로 못 누름
- **스와이프 제스처**: 라이트박스에서 좌우 스와이프로 사진 넘기기 (Phase 2)
- **핀치 줌**: 사진 확대 시 두 손가락 줌 지원 고려
- **hover 효과 주의**: 모바일에는 hover가 없음. `hover:` 스타일은 부가적으로만 사용

### 5. 업로드 (Phase 2)

- **`accept="image/*,video/*"`**: 모바일에서 카메라/갤러리 선택 UI가 자동으로 뜸
- **파일 크기 제한**: 모바일 네트워크에서 큰 파일 업로드는 타임아웃 위험. 프론트에서 미리 체크
- **업로드 진행률 표시**: 모바일 네트워크는 느릴 수 있어서 진행 바 필수
- **HEIC 포맷 대응**: iPhone은 기본적으로 HEIC로 사진을 저장. 서버에서 JPEG 변환하거나 브라우저에서 처리

### 6. 성능

- **무한 스크롤 또는 페이지네이션**: 사진이 많으면 한 번에 다 불러오면 안 됨. 20~30개씩 끊어서 로드
- **이미지 크기 `width`/`height` 명시**: 안 하면 이미지 로드될 때 레이아웃이 덜컹덜컹 움직임 (CLS)
- **Service Worker / 캐싱**: 같은 썸네일 반복 로드 방지 (나중에)

### 7. iOS Safari 특이사항

- **100vh 문제**: iOS Safari에서 `height: 100vh`가 주소창 높이를 포함해서 계산됨. `dvh` 단위 사용 권장
- **영상 자동재생 불가**: 음소거(`muted`) 상태에서만 autoplay 가능
- **`position: fixed` 키보드 이슈**: 검색/업로드 인풋에서 키보드 올라올 때 fixed 요소가 밀림
